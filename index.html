Compatibilidad con los navegadores

  • Todo eso funciona en todos los navegadores "modernos"
  • Transiciones, animaciones, tranformaciones en 3D no funcionan en IE9 :(

Soluciones

  • Usar Modernizr para detectar si el navegador puede user estes atributos y usar $.animate si no...
  • "Degrade gracefully". Los usarios de IE9 simplemente no van a ver estes efectos visuales.

AJAX

Historia

  • HTML originalmente fue designado para documentos
  • Netscape (Mozilla) agregó JavaScript
  • Podíamos modificar el DOM pero no había manera de hablar con la red

Primera solución: iframes

      var testFrame = document.createElement("iframe");
      testFrame.id = "testFrame";
      testFrame.src = "https://mejorando.la";
      document.body.appendChild(testFrame);

Historia

  • Microsoft quería hacer una aplicación para Outlook dentro del navegador. Hicieron una interfaz llamada IXMLHTTPRequest en ActiveX en IE5
  • Mozilla implementó su propría versión nativamente en el navegador, seguiendo el ejemplo de Microsoft
  • XMLHttpRequest fue copiado por todos los navegadores y se hizo un estandard "de-facto"
    • Mozilla in 2002
    • Safari 2004
    • Opera 2005
    • Microsoft (IE7) 2006

Verbos HTTP

  • GET - Dame los datos que estoy pidiendo
  • POST - Guarda estes datos
  • PUT, PATCH - Estes datos han cambiando
  • DELETE - Esos datos de ayer? Ya no los necesito
  • GET
    • Quiero los resultados de ésta búsqueda con éstos parámetros
    • Quiero ésta imagen
  • POST
    • Registrando un usuario
    • Subiendo una imagen

Same-Origin Policy

Los navegadores no permitan javascript hacer cualquiera pedido. Sólo puedes hacer un pedido al mismo dominio.

Si tu pagina es: http://example.com/index.html. Sólo puedes hacer pedidos a urls en este mísmo dominio.

    CORS - La solución mejor
